10ten Japanese Reader is available for iPhones and iPads!

The iOS version

Ever since the announcement that iOS 15 would support Web Extensions, we teamed up with Jamie Birch (LinguaBrowse on Twitter) to bring 10ten Japanese Reader to iPhones and iPads.

We ended up rewriting major parts of the extension to make it work on smartphones with small screens and less power but the biggest challenge was working out how to “hover” over text without a mouse. You could tap the text, but what if the text was part of a link?

For that Jamie had the brilliant idea of using a “puck”.

Don't have an iPhone or iPad?

Many of the changes we made to support iOS like improved lookup performance, touch screen support, and improved iframe handling should still bring some benefits to desktop devices.

Also, you can manually turn on the lookup “puck” from the options page in any browser, should that prove handy!

A little highlight

Another of the challenges we faced with supporting iOS is that 10ten indicates the matched text by selecting it, but text selections are not consistently shown on iOS. In future, we hope to use a new feature coming to Web browsers called the Highlight API to fix this. In fact, that's how we added the yellow highlights in the video above.

Using the Highlight API is better than selecting the text because it doesn't interfere with any text you might have already selected. Safari and Chrome (and Edge) already have an experimental implementation of this feature. If you'd like to try it, the instructions for enabling it are below. If you're on iOS, we think it will make using 10ten a lot easier so we encourage you to give it a try.

  • Safari on iOS (iPhone and iPad)

    Settings → Safari → Advanced → Experimental Features → Highlight API (Screenshots)

  • Safari on Mac

    1. Enable the “Develop” Menu: Safari → Preferences → Advanced → Show Develop menu in menu bar
    2. Enable the Highlight API: Develop → Experimental Features → Highlight API

  • Chrome and Edge

    Type chrome://flags/#enable-experimental-web-platform-features in the address bar, and switch “Experimental Web Platform features” to “Enabled”, then click “Relaunch”.

    Beware that enabling this might cause some websites to break.

Unfortunately Firefox does not yet have an implementation of this feature but you can follow Mozilla bug 1703961 if you would like to see when it becomes available.

Price fixing

After our last major release we got lots of great feedback including this clever suggestion:

Do you think similar support for numbers or currency would be possible in a future update? It always takes me a few seconds to parse numbers like 150億円 etc

We haven't done numbers yet (soon!) but currency conversion is here! With a bit of luck, if you have 10ten enabled, it has already downloaded the currency rates in the background and the following examples might just work:

  • 150億円
  • 9万8800円
  • ¥ 88,000.00

You can change the currency for conversion in the options page.

The currency data is updated once a day, but only once you enable 10ten. As a result, if you want up-to-the-minute conversion rates, you'll want to use a real currency converter.
